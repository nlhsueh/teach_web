# 第 1 章：Web 應用程式概觀
## (Web Overview)

建立對 Web 開發的全面認知，從基本概念到現代化開發架構。

---

## 1.1 什麼是 Web？

Web 是一個運行在網際網路之上的**應用層分散式系統**。

### 四大核心支柱
1. **URI/URL**：資源導向 (Resource-Oriented)。
2. **HTTP/HTTPS**：無狀態傳輸協議。
3. **HTML & 渲染引擎**：跨平台展示標準。
4. **超連結性**：全球性的有向圖架構。

---

## 深入理解「無狀態 (Stateless)」

- **定義**：伺服器不會「記住」之前的請求。每一次請求都是全新的。
- **生活類比**：速食店點餐。除非出示「收據」或「會員卡」，否則店員不認得你。
- **解決方案**：
    - **Cookies / Sessions**
    - **Token (JWT)**
    - 補足系統的「有狀態」互動感。

---

## Web 的三維演進

| 特徵 | **Web 1.0** (唯讀) | **Web 2.0** (互動) | **Web 3.0** (智慧) |
| :--- | :--- | :--- | :--- |
| **數據流向** | 單向 | 雙向交互 | 所有權與理解 |
| **技術特徵** | 靜態 HTML | 資料庫 / API | 區塊鏈 / AI |
| **核心** | 文件檢索 | 應用服務 | 智慧自主 Agent |

---

## 為什麼 Web 開發如此盛行？

- **跨平台與低門檻**：只要有瀏覽器，無視作業系統。
- **即時發佈與更新**：修改程式碼，重啟伺服器，全世界同步更新。
- **統一的生態系**：由開放標準 (W3C) 定義，無廠商壟斷。

### 2025 最新統計
- 全球約 **55-60 億** 上網人口 (佔全球 ~70%)。
- **JavaScript** 仍是網頁開發首選。

---

## 1.2 前後端分離架構

現代 Web 多採用「解耦架構 (Decoupled Architecture)」。

- **前端 (Front-End)**：「面子」— 使用者看到的、互動的。
- **後端 (Back-End)**：「裡子」— 邏輯、資料庫、安全性。

---

## 前端開發 (Front-End)

**核心職責**：UI 設計、UX 體驗、響應式佈局 (RWD)。

### 三大支柱
1. **HTML (骨架)**：結構定義。
2. **CSS (皮肉)**：視覺風格與版面。
3. **JavaScript (靈魂)**：動態互動與邏輯。

---

## 後端開發 (Back-End)

**核心職責**：資料處理、商業邏輯、保護隱私。

### 關鍵技術
- **語言**：Python (Django), Node.js, Java (Spring), Go 等。
- **資料庫**：
    - **SQL**: MySQL, PostgreSQL (關聯式)。
    - **NoSQL**: MongoDB (非關聯式)。

---

## 為什麼要「前後端分離」？

1. **專業分工 (Specialization)**：前端專注渲染與 UX，後端專注併發與資料。
2. **多端共用 (One Backend, Multiple Clients)**：同一 API 支援網頁、App 與 IoT。
3. **獨立部署與擴展 (Scalability)**：針對瓶頸獨立擴充伺服器。

---

## Web 前端的渲染策略

1. **CSR (客戶端渲染)**：
    - 伺服器給「空殼 + JS」。
    - 優點：SPA 體驗，頁面切換快。
2. **SSR (伺服器端渲染)**：
    - 伺服器給「完整的 HTML」。
    - 優點：**SEO 極佳**，首屏載入感快。
3. **SSG (靜態生成)**：
    - 編譯時產生成千上萬 HTML。
    - 優點：最高效能、最安全。

---

## 1.3 網頁框架 (Frameworks)

### 前端 CSS 框架
- **Bootstrap**：組件導向 (btn, card)，開發極快。
- **Tailwind CSS**：工具導向 (Utility-first)，高度自訂化。

### 後端框架
- **Django**：Python 界的全功能框架 (MVT 模式)。
- **Express / Spring / Laravel**。

---

## 1.4 現代開發工具與 DevOps

- **套件管理**：npm, pip。
- **建置工具**：Vite, Webpack。
- **容器化**：**Docker** (確保環境一致性)。
- **CI/CD**：自動測試、自動部署。
- **基礎設施即程式碼 (IaC)**：雲端資源管理。

---

## 1.5 什麼是「好的」網頁系統？

1. **使用者友善介面**：降低認知負荷。
2. **快速反應性**：TTFB 優化、零挫折感。
3. **行動裝置友善 (RWD)**：Mobile-First。
4. **一致性**：建立操作慣性。
5. **無障礙環境 (A11y)**：Alt 標籤、語意化 HTML。
6. **友善的錯誤處理**：化解危機而非報錯。

---

## 1.6 常見名詞解釋 (Glossary)

- **DOM**：JS 操控網頁的橋樑。
- **SPA**：單頁應用 (不跳轉刷新)。
- **SEO**：搜尋引擎優化。
- **JSON**：主流資料交換格式。
- **API**：系統間溝通的「菜單」。
